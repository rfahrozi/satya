Ringkasan Arsitektur & File Utama

Dokumen ini mendeskripsikan struktur dan alur kerja proyek Single-Page Application (SPA) React yang digunakan untuk portal laporan dan manajemen satuan kerja.

Struktur Modul Proyek:

  • Entry / routing: App.jsx
  • Shell/layout: components/Layout.jsx
  • Halaman: pages/Login.jsx, Dashboard.jsx, SatkerPortal.jsx, UserManagement.jsx
  • Layanan HTTP: src/lib/axios.js
  • Styling: src/index.css (Tailwind + custom utilities)
  • Pola state/side-effects: react (hooks) + @tanstack/react-query untuk fetching/mutasi dan invalidasi cache.
  • Ikon & UI primitives: lucide-react (ikon), Tailwind utility classes, CSS kustom (.btn, .card, .glass-header).

Komponen dan Tanggung Jawab Utama

App.jsx

  • Menyediakan Router utama (react-router-dom).
  • Mengimplementasikan wrapper otorisasi: ProtectedRoute yang membaca localStorage untuk token dan user, mengarahkan ke /login bila tidak terautentikasi, dan memeriksa role untuk akses (redirect fallback ke /portal atau /dashboard bila tidak berhak).
  • Mendaftarkan rute: /login, /dashboard, /users, /portal, plus 404 fallback.

Layout.jsx (App Shell)

  • Render top navigation bar bergaya glassmorphism; menampilkan logo, nama user, per-role nav links (kondisional berdasarkan user.role), dan tombol logout.
  • Menyisipkan <Outlet /> untuk merender halaman rute anak.
  • Logout menghapus localStorage dan mem-navigate ke /login.

Login.jsx

  • Form login sederhana. Mengirim POST ke endpoint auth via axios global.
  • Menyimpan satya_token dan satya_user di localStorage setelah login sukses, lalu redirect sesuai role (SATKER_PN → /portal ; lainnya → /dashboard).
  • Menangani status loading, menampilkan error pesan dari response backend.

Dashboard.jsx

  • Menampilkan agregat statistik, filter bulan/tahun, pencarian satker.
  • Memanfaatkan react-query (useQuery) untuk mengambil data dashboard agregat (/reports/dashboard-agregat).
  • Menyediakan aksi admin (verifikasi): open modal, kirim patch via useMutation ke /reports/:id/verify, lalu invalidate query untuk refresh data.
  • Fitur download menggunakan endpoint /reports/:id/download yang mengembalikan URL file lalu membuka di tab baru.
  • Render tabel dinamis berisi satker → tiap satker berisi daftar laporan (status upload, ketepatan waktu, status verifikasi). UI conditional menampilkan tombol Aksi jika user adalah admin.

SatkerPortal.jsx

  • Ditujukan bagi role SATKER_PN: menampilkan grid dokumen wajib (report cards) per periode.
  • Upload flow: buka modal, pilih file → buat FormData → posting ke /reports/upload via useMutation (multipart). Setelah sukses, invalidasi cache my-progress.
  • Badge styling berdasarkan status_verifikasi, status_ketepatan_waktu, submission_id.

UserManagement.jsx

  • CRUD ringan: fetch users (/auth/users) dan fitur pembuatan akun satker via POST /auth/users.
  • Modal pendaftaran, form validation minimal di UI, react-query untuk fetch & mutate.

src/lib/axios.js (HTTP service)

  • axios instance dengan baseURL http://localhost:3000/api/v1 dan Content-Type JSON.
  • Request interceptor: otomatis menyisipkan header Authorization: Bearer ${token} dari localStorage.
  • Response interceptor: langsung mengembalikan response.data (memudahkan pemakaian), dan menangani 401 dengan membersihkan localStorage lalu redirect ke /login.

Kumpulan Utilitas & Styling

  • Tailwind CSS: index.css mengimpor Google Font Inter, meng-@import tailwind, dan menambahkan layer components untuk .btn, .card, .glass-header, serta beberapa variabel tema custom.
  • Animasi ringan: .animate-fade-in, shimmer/skeleton digunakan untuk loading placeholders.
  • Komponen UI konsisten memakai utility classes Tailwind dan class helper kustom.

Dependensi Eksternal Signifikan

  • react + react-router-dom (routing)
  • @tanstack/react-query (data fetching, caching, invalidation, mutations)
  • axios (HTTP client)
  • lucide-react (ikon SVG)
  • tailwindcss (styling utilities)

Alur Eksekusi End-to-End

  1. Inisialisasi aplikasi: Browser mengunjungi root; Router di App.jsx mem-redirect ke /login bila path '/'. Jika token sudah ada, ProtectedRoute mengecek token & user di localStorage terlebih dahulu.
  2. Login: User submit form di Login.jsx. Jika sukses, simpan token & user ke localStorage, lalu navigasi halaman sesuai role.
  3. Akses halaman terlindungi: Layout.jsx dibuka. Layout membaca user dari localStorage dan menampilkan menu sesuai role. Outlet merender halaman spesifik.
  4. Data fetching & display: Komponen menggunakan useQuery untuk memanggil API. Interceptor mengekstrak data, komponen merender UI (contoh: tabel dinamis di Dashboard).
  5. Mutasi & side-effects: Saat melakukan aksi (verifikasi/upload), useMutation dipanggil. Pada onSuccess, query di-invalidate agar data segar ditarik kembali.
  6. HTTP lifecycle global: Request otomatis menyertakan token. Jika merespon 401, session dihapus dan user dipaksa redirect ke /login.

Key Asumsi & Kontrak Backend

  • API mengembalikan objek konsisten: { success, message, data }.
  • Otentikasi berbasis JWT (token dikirim di header Authorization).
  • Endpoint penting terdefinisi dengan jelas (auth, reports, users, upload).

Catatan Teknis & Rekomendasi

  • Penyimpanan token di localStorage praktis tapi rentan XSS. Pertimbangkan HttpOnly cookie untuk keamanan lebih tinggi jika memungkinkan.
  • Pengekstrakkan response.data di interceptor harus konsisten agar tidak merusak komponen jika format response backend berubah.
  • Validasi file upload masih minimal di UI; pastikan validasi dan sanitasi ketat di sisi backend.
  • Aksesibilitas (A11y) perlu audit tambahan (keyboard focus, labels, kontras warna).

Kesimpulan

Aplikasi terstruktur rapi mengikuti pola SPA modern dengan routing terpusat, app shell, perlindungan rute berbasis role, lapisan layanan axios dengan interceptors, dan lapisan data menggunakan react-query untuk sinkronisasi state server yang efisien.
