Panduan Deployment Sistem SATYA

Sistem Administrasi dan Tata kelola Yudisial yang Akuntabel | (April 2026)

1. Prasyarat Sistem (Prerequisites)

Aplikasi SATYA dibangun dengan arsitektur modern (*containerized*). Pastikan VPS Anda Ubuntu 22.04+ telah terpasang komponen berikut:

  • Docker Desktop / Docker Engine: Tool utama (wajib).
  • Docker Compose: Untuk menjalankan Node API, Worker, PostgreSQL, Redis, dan MinIO dalam satu instruksi.
  • Nginx: Sebagai Reverse Proxy dan SSL Manager di VPS produksi.

2. Konfigurasi Lingkungan (.env)

Buat salinan atau buat file .env di root folder aplikasi dan lengkapi kredensial secara ketat:

PORT=3000 NODE_ENV=production # Database Credentials DB_HOST=db DB_USER=satya_admin DB_PASS=PasswordPostgresProduksi DB_NAME=satya_db # JWT & Security JWT_SECRET=RandomSecureKeyLebihDari32Karakter JWT_EXPIRES_IN=1d # MinIO Storage (Object Upload) MINIO_ENDPOINT=minio MINIO_PORT=9000 MINIO_ACCESS_KEY=minio_admin MINIO_SECRET_KEY=minio_secure_password # Redis Credentials REDIS_HOST=redis REDIS_PORT=6379 # Layanan SMTP / Email Pengingat Otomatis SMTP_HOST=smtp.gmail.com SMTP_PORT=587 SMTP_USER=email.resmi.ptkepri@gmail.com SMTP_PASS=PasswordAplikasiGmailAnda DEADLINE_DAY=10

3. Proses Build & Deployment dengan Docker

SATYA telah dibungkus menggunakan Docker Compose, yang akan mengeksekusi service Database, Redis Cache, Storage Minio, Web API, dan Job Worker Scheduler email secara bersamaan.

Membangun dan Menjalankan Aplikasi di Background:

docker-compose --env-file .env up --build -d

Menjalankan Database Migration & Seeding:

PENTING! Lakukan inisiasi skema database dan Inject *seed data* SATYA wajib sekali saja saat deploy pertama kali.

docker-compose exec app npx knex migrate:latest docker-compose exec app npx knex seed:run
Catatan: Data seed awal mencakup Akun Admin (admin_pt), Akun Pimpinan (pimpinan_pt), Akun contoh Satker, serta referensi 28 tipe Laporan Wajib pengadilan.

4. Konfigurasi Nginx (Reverse Proxy)

Nginx meneruskan request dari domain/publik (https) ke port internal lokal 3000 (app).

sudo nano /etc/nginx/sites-available/satya.pt-kepri.go.id

Tempelkan konfigurasi berikut:

server { server_name satya.pt-kepri.go.id; # Batasan Upload proxy pass Nginx selaras dengan Multer App (10MB Limit) client_max_body_size 12M; location / { proxy_pass http://localhost:3000; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; } }

Buat symlink dan restart nginx:

sudo ln -s /etc/nginx/sites-available/satya.pt-kepri.go.id /etc/nginx/sites-enabled/ sudo nginx -t sudo systemctl restart nginx

5. Keamanan SSL (HTTPS)

Gunakan Certbot untuk mengamankan komunikasi data ke API. Endpoint Authentication rate limit juga akan dilindungi HTTPS.

sudo apt install certbot python3-certbot-nginx -y sudo certbot --nginx -d satya.pt-kepri.go.id
PENTING: PERAWATAN MINIO STORAGE
Seluruh Laporan format PDF/XLSX akan masuk ke service `minio`. Volume docker untuk container MinIO dan container PostgreSQL mutlak WAJIB di-*backup* (Cron job `tar` atau sinkronisasi rsync) secara rutin untuk mencegah *Data Loss* yang fatal.
