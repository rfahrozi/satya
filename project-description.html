📊 Deskripsi Lengkap Proyek MONEV SATYA

🎯 Overview Proyek (Production Ready)

SATYA (Sistem Administrasi dan Tata kelola Yudisial yang Akuntabel) adalah sistem Monitoring dan Evaluasi berbasis web full-stack yang dirancang khusus untuk Pengadilan Tinggi Kepulauan Riau (PT Kepri).

Aplikasi ini memungkinkan pimpinan untuk memantau kepatuhan Satuan Kerja (Satker) Pengadilan Negeri di wilayah Kepri secara real-time, dilengkapi dengan fitur unggah laporan, verifikasi, rekapitulasi heatmap, hingga notifikasi email otomatis.

Tech Stack: Node.js/Express + React/Vite + PostgreSQL + MinIO + Redis + Docker + Nginx

📈 Fitur Sistem Utama V1.0.0

  • Multi-tenant (Satker Isolation): Setiap Satker memiliki ruang kerja yang terisolasi dan aman.
  • Dashboard Pimpinan & Heatmap: Visualisasi persentase kepatuhan (Hijau, Kuning, Merah) secara langsung.
  • Object Storage Berkeamanan Tinggi (MinIO): Penyimpanan dokumen aman menggunakan proxy stream internal untuk menghindari public exposure.
  • Sistem Antrean Email (BullMQ) & Brevo API: Notifikasi dan pengingat dikirim menggunakan antrean Redis dan Brevo API untuk menembus firewall VPS yang memblokir port SMTP.
  • Arsitektur Nginx Reverse Proxy: Routing base_path dinamis (`/satya`) untuk hidup berdampingan dengan proyek lain di VPS Ubuntu tanpa bentrok.
  • Keamanan Zero Trust: Database, Redis, dan MinIO beroperasi 100% di dalam internal network Docker tanpa mapping port ke host.
  • Landing Page Modern: UI/UX elegan dengan animasi menggunakan `framer-motion` dan TailwindCSS.

📋 Penjelasan Arsitektur Baru

1. Proxy MinIO & Keamanan Docker

Demi keamanan, port MinIO tidak diekspos ke publik VPS. Sebagai gantinya, aplikasi Node.js akan mem-proxy file tersebut langsung ke pengguna. Frontend akan memanggil `/api/v1/reports/proxy?url=...` dan Node.js akan mengalirkannya secara aman (Stream).

2. Nginx & Base Path `/satya`

Proyek diatur agar tidak mengganggu sistem lain di server (seperti ELID). Nginx akan me-reverse proxy trafik `/satya/` menuju Node.js. Node.js telah dikonfigurasi untuk menyesuaikan `process.env.BASE_PATH` agar tidak terjadi 404 Not Found.

3. Integrasi Brevo API

Mengingat provider VPS sering memblokir port SMTP standar, layanan pengiriman email telah beralih menggunakan API HTTP POST ke Brevo. Jika `BREVO_API_KEY` tersedia, sistem otomatis memutus SMTP dan menggunakan HTTP.

🚀 Cara Menjalankan (Deployment)

# 1. Pastikan File .env sudah ada (dengan BREVO_API_KEY)

# 2. Build dan Jalankan Container
docker-compose -f docker-compose.prod.yml up -d --build

# 3. Inisialisasi Database (Hanya Sekali)
docker exec -it monev_app npm run migrate:latest
docker exec -it monev_app npm run seed:run
